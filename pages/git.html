Git

Es una terminal la cual se puede instalar en nuestro dispositivo y luego podemos crear copias para subirlas al repositorio

creando un repositorio

  • git --version

    Muestra la versión de Git instalada en tu sistema.

  • git config --global user.name "Tu Nombre"

    Establece el nombre de usuario que se utilizará para las confirmaciones de commit.

  • git config --global user.email "tu_email@example.com"

    Establece el correo electrónico que se utilizará para las confirmaciones de commit.

  • git config --list

    Muestra la configuración actual de Git para el repositorio y comprobamos si guardamos bien el usuario.

  • git init

    Inicializa un repositorio Git en el directorio actual, esto solo se hace la primera vez si no hay ninguna carpeta de git en nuestro proyecto.

  • git add .

    Agrega un archivo al área de preparación para ser incluido en el próximo commit. la parte del "." podemos remplazarla por el archivo específico que queremos que guarde.

  • git commit -m "mensaje"

    Guarda los cambios realizados en el repositorio localmente.

  • git branch

    Muestra todas las ramas en el repositorio local.

  • git branch <nueva-rama>

    Crea una nueva rama en el repositorio.

  • git checkout <rama>

    Cambia a la rama especificada.

  • git branch -M main

    Renombra la rama actual a 'main'.

  • git remote add origin <url>

    Conecta tu repositorio local con un repositorio remoto en la URL especificada que nos dará GitHub.

  • git push -u origin main

    Sube los cambios locales a la rama principal del repositorio remoto y establece la conexión entre la rama local y la rama remota.

  • git status

    Muestra el estado actual del repositorio, incluyendo los archivos modificados, los archivos en el área de preparación y los que no están siendo rastreados por Git.

  • git branch

    Muestra todas las ramas en el repositorio local.

  • git checkout <rama>

    Cambia a la rama especificada.

  • git add .

    Agrega un archivo al área de preparación para ser incluido en el próximo commit. la parte del "." podemos remplazarla por el archivo específico que queremos que guarde.

  • git commit -m "mensaje"

    Guarda los cambios realizados en el repositorio localmente.

  • git push origin <rama>

    Sube los cambios locales a un repositorio remoto en la rama especificada.

  • git log

    Muestra un registro de todos los commits realizados en el repositorio.

uniendo ramas

Primero debemos posicionarnos sobre la rama que queramos y llamar desde esta la rama que queremos fusionar a la que estamos

  • git checkout <rama>

    Cambia a la rama especificada.

  • git merge <otra-rama>

    Une la rama actual con otra rama especificada.

  • git push origin --delete <rama>

    Eliminar la rama del repositorio remoto.

Trabajando desde otra pc

  • git clone <URL-del-repositorio>

    Clonar el repositorio en tu computadora portátil.

  • git add .

    Agrega un archivo al área de preparación para ser incluido en el próximo commit. la parte del "." podemos remplazarla por el archivo específico que queremos que guarde.

  • git commit -m "mensaje"

    Guarda los cambios realizados en el repositorio localmente.

  • git push origin <rama>

    Sube los cambios locales a un repositorio remoto en la rama especificada.

  • git pull origin <rama>

    Descargar los cambios desde el repositorio remoto y actualizar tu copia local desde la otra computadora.

  • git checkout --track origin/<rama>

    Trae a la fuerza una nueva rama en caso de que no la tengamos

Volver un commit atras

  • git reset --hard <hash>

    Sirve para volver un commit para atras, usariamos el comando git log y de ahi sacariamo el hash de el commit que queramos
