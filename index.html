QUE ES HTML

Es la base del desarrollo web, le da la estructura, vendría a ser el esqueleto de la web, este tiene muy poca apariencia por eso luego usamos css. por ende con html no quedaria bonito. Su nomeclatura por default al crear el archivo es index.html

PAGINA Y SITIO WEB

  • Pagina Web

    Una página web es un documento individual en la web que se puede acceder mediante un navegador web.

  • Sitio Web

    Un sitio web es un conjunto de páginas web relacionadas y vinculadas entre sí bajo un dominio específico. Un sitio web puede incluir varias páginas que están diseñadas y organizadas para ofrecer información coherente o servicios relacionados.

CONSEJOS

  • Uso de Caracteres Especiales:

    Evita el uso de caracteres especiales, tildes o símbolos raros en los nombres de archivos, carpetas y recursos web. Opta por nombres simples y con caracteres alfanuméricos.

  • Comentarios Significativos:

    Utiliza comentarios HTML de manera efectiva para explicar secciones de código. Esto facilita la comprensión del código para ti y otros desarrolladores que puedan revisarlo.

  • Indentación y Organizació:

    Mantiene una buena estructura de indentación y organiza tu código de manera clara. Esto mejora la legibilidad y facilita el mantenimiento del código a largo plazo.

  • Peso del Código:

    Recuerda que cada imagen que pongas en tu codigo podria hacer que este pese mas haciendo lenta la pagina. usa aplicaciones que las optimicen para pesar lo menos posible.

  • Accesibilidad:

    Diseña y desarrolla tu contenido web pensando en la accesibilidad. Utiliza atributos como alt en imágenes

METADATOS

  • <html>

    Representa la raiz de un documento HTML. El resto de elementos descienden de este elemento.

  • <base>

    Especifica la dirección URL base que se utilizará para todas las direcciones URL relativas contenidas dentro de un documento.

  • <head>

    Provee información general (metadatos) acerca del documento, incluyendo su título y enlaces a scripts y hojas de estilos.

  • <link>

    Especifica la relación entre el documento actual y un recurso externo. Los usos posibles de este elemento incluyen la definición de un marco relacional para navegación. Este elemento es más frecuentemente usado para enlazar hojas de estilos.

  • <meta>

    Re utiliza para proporcionar información adicional sobre el documento HTML. Esta etiqueta no tiene contenido visible en la página, pero juega un papel importante en la configuración de la información del documento para el navegador y otros agentes de usuario.

  • <style>

    Se utiliza para poner estilos directamente en el html. sin embargo lo recomendable es tenes una hoja aparte que sea de estilos o css

  • <ticle>

    Define el título del documento que se muestra en un browser la barra de título o la pestaña de una página.

  • <body>

    Representa el contenido de un documento HTML. Solo puede haber un elemento <body> en un documento.

SEMANTICA HTML y CONTENEDORES

  • <header>

    Define la sección de encabezado de un documento o de una sección específica. Puede contener elementos como títulos, logotipos o enlaces de navegación.

  • <nav>

    Representa una sección de navegación que contiene enlaces a otras páginas o recursos dentro del sitio web. solo se usa uno por pagina

  • <main>

    Define el contenido principal de un documento. Debería evitar incluir encabezados, pies de página o barras laterales dentro de este elemento.

  • <article>

    Representa un contenido independiente y autónomo, como un artículo, una entrada de blog o un comentario. Debería ser reutilizable y distribuible de manera independiente.

  • <section>

    Define una sección genérica en un documento. Puede contener contenido temáticamente relacionado y puede usarse dentro de cualquier etiqueta semantica.

  • <aside>

    Define contenido relacionado pero secundario, como barras laterales, anuncios o información adicional no central para el contenido principal.

  • <footer>

    Define la sección de pie de página de un documento o de una sección específica. Puede contener información de derechos de autor, enlaces de contacto u otros elementos de cierre.

  • <table>

    Define la estructura de la tabla. Puede contener otras etiquetas como <thead>, <tbody> y <tfoot> para organizar la tabla en secciones.

  • <div>

    Es un contenedor de bloque utilizado para agrupar, el cual ocupa todo el re no tiene peso semantico es decir vale menos que todos los contenedores dichos anteriormente.

  • <span>

    Es un contenedor en linea utilizado para agrupar, el cual no tiene peso semantico es decir vale menos que todos los contenedores dichos anteriormente.

AGREGAR IMAGENES y OTROS

  • <img>

    Una etiqueta muy simple donde dentro del atributo src escribiremos la direccion de la imagen. Tambien es importante que dentro del atributo alt escribamos una descripcion breve de la imagen.

  • <video>

    Una etiqueta muy simple donde dentro del atributo src escribiremos la direccion del video.

  • <audio/audio>>

    Una etiqueta muy simple donde dentro del atributo src escribiremos la direccion del audio .

  • <iframe>

    Es un código html que se consigue desde el video en youtube por ejemplo o google maps al tocar compartir y luego insertar. Entonces nos da el código, el cual debemos copiar y pegar en el html

CREAR TABLA

La tabla tiene su propia etiqueta semantica la cual es <table>.

  • <thead>

    define la sección de encabezado de la tabla de forma semantica.

  • <tbody>

    define la sección principal de datos de la tabla de forma semantica.

  • <tfoot>

    Define la sección de pie de tabla de forma semantica.

  • <th>

    Define una celda de encabezado. Se utiliza para las celdas en la primera fila o columna para representar encabezados.

  • <tr>

    Define una fila de la tabla.

  • <td>

    Define una celda de datos. Se utiliza para el contenido de las celdas
